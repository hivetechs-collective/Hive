Memory Query Guide

Overview

The Hive Consensus memory system provides intelligent context retrieval for all your AI interactions. It maintains a unified database of all conversations, learnings, and insights.

Query Commands

Universal Commands (work across all CLI tools):

hive-memory context "What have we worked on?"
hive-memory query "Show me Python examples"
hive-memory contribute solution python "Fixed import issue"

Natural Language Triggers

These phrases automatically trigger memory queries:

  • "What did we..." - Retrieves past conversations
  • "Show me our..." - Displays previous work
  • "Remind me about..." - Recalls specific topics
  • "What have we..." - Shows work history

Tool-Specific Shortcuts

hive-memory-claude-code context "test query"
hive-memory-gemini-cli context "test query"
hive-memory-grok context "test query"

Database Access Guide

Unified Database Location

All data is stored in: ~/.hive/hive-ai.db

This SQLite database contains all conversations, memory, settings, and analytics.

Direct Query Examples

Natural Language Queries:

"Query .hive-ai.db: What have we worked on with authentication?"

Named Views:

"Query .hive-ai.db view 'recent_work'"
"Query .hive-ai.db view 'solutions' about React"

Smart Context Function:

"Query .hive-ai.db: SELECT get_context('debugging TypeScript')"

Direct SQL (Advanced):

"Query .hive-ai.db: SELECT * FROM messages WHERE content LIKE '%API%' LIMIT 5"

Available Views

  • recent_work - Last 100 assistant responses
  • solutions - Successful problem resolutions
  • errors_fixed - Resolved error messages
  • code_examples - Code snippets by language
  • project_context - Current project information

Database Statistics

# Check memory size
"Query .hive-ai.db: SELECT COUNT(*) FROM messages"

# View usage over time
"Query .hive-ai.db: SELECT DATE(created_at) as day, COUNT(*) FROM messages GROUP BY day"

CLI Tools Integration

Supported Tools

  • Claude Code - Full memory integration via Direct API
  • Grok CLI - Morph Fast Apply with memory context
  • Gemini CLI - Unified command support
  • aichat - Multiple model access with memory
  • mods - Streaming responses with context
  • Cursor Composer - IDE integration

Memory Service API

Endpoints:

POST /api/v1/memory/query      - Query memories with context
POST /api/v1/memory/contribute - Share new learnings
GET  /api/v1/memory/stats      - Memory statistics
GET  /api/v1/memory/tools      - List connected tools

Installation & Configuration

Tools are automatically configured when launched from Hive Consensus. The Memory Service provides:

  • Automatic port discovery (3457-3560 range)
  • Token-based authentication
  • Tool-specific identification
  • Unified command system

Enterprise Features

Advanced capabilities for professional use:

  • Hooks system for custom workflows
  • Approval workflows for sensitive operations
  • Audit logging for compliance
  • Custom memory retention policies
